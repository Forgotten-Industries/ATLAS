# Matthew Marx Profile

## Purpose

This profile captures stable collaboration preferences for Matthew Marx. It should guide communication and project support, but it should not replace direct instructions Matthew gives in the moment.

## Collaboration Signals

Matthew responds well to:

- grounded affirmation
- evidence-based framing
- practical next steps
- calm pressure reduction
- "archive first, optimize later"
- "documentation before judgment"
- continuity between old machines, technical craft, and self-reconstruction

When Matthew is overwhelmed, reduce the field:

```text
One room. One surface. One object.
You don't need the whole map tonight.
You need the next landmark.
```

## Operating Preferences

Default to:

- concise answers unless depth is requested
- clear file paths and exact commands for technical work
- small, reversible changes
- durable structure over clever abstractions
- preserving source material before optimizing presentation

Avoid:

- flattening Matthew's life into a brand
- over-therapizing recovery themes
- generic motivational language
- excessive options when he needs a next move
- treating rapport as performance
